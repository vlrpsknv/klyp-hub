Продукт — Позиционирование
НЕ микродрамы — не азиатский формат DramaBox/ReelShort. Сериальный контент — связанные шорты, не рандомный фид.
Креатор-центричная платформа — креатор создаёт → публикует → шарит в соцсетях → привлекает зрителей на стриминг → зарабатывает.
Две рефералки: креатор → другие креаторы, креатор → зрители.
Документ по миссии — подъедет скоро, даст больше контекста.
Продукт — Publishing Console
Библиотека тайтлов = главный экран. Все тайтлы с минимальной аналитикой. Клик → детали, просмотры, как видят в фиде.
Детальная аналитика — отдельный раздел. Best performing — самые успешные тайтлы, зрители/месяц, пиковая аудитория, охват.
Video Editor — третий по приоритету раздел.
Загрузка своего контента — без доп. оплаты. Нужна только подписка на паблишинг. Выбор серий, расписание, бесконечная догрузка.
Шеринг в соцсети — кнопка публикации в Instagram/TikTok — основной движок привлечения зрителей на стриминг.
Кабинет заработков — earnings dashboard. Детали не обсуждались.
Реферальная система — две рефералки (креаторы + зрители). Не на главной, но обязательная фича.
Аналогии: YouTube Studio, Instagram Studio, Google Play Console, App Store Console, OnlyFans (без adult).
Продукт — Video Editor
Layout: контент по центру (вертикально), окно промтинга / управления справа.
Вдохновение: Sims character editor — RPG-стиль: объект по центру, панель настроек сбоку. НЕ prompt-first с большой строкой внизу.
Файл-шеринг между AI = ГЛАВНЫЙ UX-дифференциатор. Файлы на серверах → бесшовный pipeline — цепочка обработки через несколько AI-сервисов без скачивания/загрузки.
Два уровня пользователей: новичок (guided path — пошаговый путь с подсказками, готовые сцены, предложенные промты) и про (full customization на каждом шаге). Один интерфейс.
Этапность: процесс разбит на этапы. Каждый этап = действие с файлом. Видишь файл + этап + действие.
Альтернатива промт-инпута: может быть по центру, незаметный, разворачивается в dev-mode. Оба варианта допустимы.
Продукт — Языки и субтитры
Субтитры по дефолту — включены везде, генерируются нейронкой.
Зритель выбирает только язык — не отдельно звук + субтитры. Одно поле.
10-20 бесплатных языков при создании. Дополнительные за деньги. Создание на любом языке.
Обложка: картинка (отдельно) + текст (отдельно, переводится под язык зрителя). Пайплайн создания обложки — отдельный процесс.
Техническое: видео хранится отдельно от аудио/субтитров. Накладываются при воспроизведении. Заложить в процесс с самого начала.
Продукт — Монетизация
Подписка на паблишинг — основная, даёт право публиковать.
Токены за редактор — внутренняя валюта за использование AI-инструментов. Через подписки или прямую покупку.
Загрузка своего контента — только подписка, без доп. чаржа.
Дизайн-решения
Double Diamond — процесс: исследование → сужение → прототип → итерации. Одобрен.
Без брендинга на MVP. Функция > визуал. Нет дизайн-системы, создавать по ходу.
Токенизация цвета — один параметр меняет весь цвет интерфейса. Ч/Б прототипы + цветовые варианты.
Избегать токсичных цветов (зелёный Hixfield = плохо). Ориентир — кино: Netflix (красный), Кинопоиск (рыжий).
Прототип формирует бизнес-требования — требования не финальные, легче формировать глядя на прототип.
Идеи предлагать сразу — инициатива приветствуется.
Прототип стриминга — пришлют для оценки, дать фидбэк. Не привязываться к его брендингу.
Цвет = data-driven — привязка к кино, решение позже когда больше данных.
Конкуренты и референсы
Syntax AI — популярен среди СНГ-креаторов, визуально приличный, пропихивают в гайдах.
Hixfield — большой на рынке, привычный. UI "казиношный матрас". Зелёный цвет = плохо. Но можно набрать функциональных идей.
Mira — пример центрального контента + боковые контролы (упомянут Ярославом).
Bananas, Citri — AI-генерация, опционально пробовать заменять компоненты, смотреть quality/performance.

6-Stage Pipeline (MVP)
1. Character/Asset Setup
Character Profiles — сохранённые персонажи для consistency в серии. Новичок выбирает из готовых, про создаёт с нуля.
2. Scene Composition
Ingredients drag & drop — перетаскивание элементов: character + object + background. Как Pika Pikascenes.
3. AI Generation
Multi-model selection — выбор лучшей AI-модели под задачу. Kling для motion, Runway для стиля, Fast для превью.
4. Refinement
Parametric controls — ползунки и дропдауны: camera, motion, lighting. RPG-стиль, как в Sims.
5. Audio/Voice
Generation + sync + multilingual. 10-20 языков. Субтитры автоматически.
6. Assembly & Export
Прямо в Library для публикации. Обложка: image + переводимый текст.
Beginner/Pro — один UI, два режима
Паттерн Capsules (Adobe Project Graph): профи строит pipeline → упаковывает в Recipe → новичок использует Recipe как простую форму.

Beginner (Guided): AI Director выбирает модели и настройки. Каждый stage = одно действие. Template-based.
Pro (Full Control): все parameters visible. Manual model selection. Custom pipeline.
Toggle: кнопка "Guided / Full Control" в top bar. Beginner видит simplified subset того же canvas.

Как это работает
Архитектура, обновление, масштабирование
Архитектура
Этот дашборд — статичный HTML-файл с встроенными данными. Он НЕ подтягивает данные автоматически из markdown. Данные захардкожены внутри HTML.

Source of truth — единственный источник правды — это markdown-файлы:
• Созвоны/ — протоколы встреч
• Requirements/ — продуктовые требования
• ACTION-ITEMS.md — задачи

Obsidian Dashboard — отдельный файл Dashboard.md в Obsidian. Он использует Dataview-запросы и подтягивает данные автоматически из YAML frontmatter — метаданные в начале каждого markdown-файла. Создал новую заметку → она сразу в таблице.
Как обновлять
После созвона:
1. Скажи Claude обработай созвон + вставь транскрипт
2. Claude создаст markdown в Созвоны/
3. Claude обновит Obsidian
4. Скажи обнови дашборд → Claude перегенерирует этот HTML

Новая задача:
• Добавляется в ACTION-ITEMS.md и в этот HTML при обновлении

Obsidian (автоматически):
• Создаёшь заметку по шаблону (Ctrl+N → Templater) → она сразу в Dashboard.md
• Нужны плагины: Dataview, Templater, Homepage
Как масштабируется
Созвоны: каждый новый созвон → новый пункт в сайдбаре + раскрывающаяся карточка на странице "Созвоны". Не отдельная HTML-страница — всё в одном файле с навигацией.

Требования: новые REQ-* карточки добавляются в грид. При росте до 30+ можно добавить фильтрацию и поиск.

Action Items: список растёт вниз, группировка по срочности сохраняется.

Когда файлов станет 50+: переход на Astro — генератор статических сайтов, автоматически читает markdown или MkDocs Material — генератор docs-сайтов из markdown. Те же markdown-файлы, но автогенерация без ручного обновления.
